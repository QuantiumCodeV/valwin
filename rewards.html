How it works?

Server and Client Seed Generation:

The game generates a server seed, which is hashed and displayed to the player before the game starts so that the player knows the server seed was created prior to making their choices.
The player can either choose their own client seed or use a random one generated by the system.

Mine Placement Using Seeds:

Once the client seed and server seed are combined (for example, using the HMAC method), a random result is generated, determining the location of the mines on the field.

Using the Provably Fair algorithm, the player can recompute the hash and verify that the mine placement was randomly generated and remained unchanged after the game started.

Fairness Verification:

After the game, the player can use the available server seed hash and client seed to recalculate the results and confirm that the game was not tampered with.
Most Provably Fair sites provide a simple interface for verification, where the player enters the initial parameters (seeds and hash) to ensure the game was fair.

Example of Provably Fair for Mines

For example:

Let’s say the server seed is "abcd1234" and the client seed is "user5678." The hash of both seeds can generate numbers indicating which cells contain mines.
After the game, the player can recalculate this result and confirm that the hash matches and the result was not tampered with.
